> **작성일:** 2026-03-23  
> **작성자:** 조정민  
> **상태:** 초안

## 1. 배경과 목표
- 기존의 html 과 javascript 만으로 이루어진 포트폴리오를 next + react + typescript + tailwind 를 기반으로 한 반응형 포트폴리오를 만들기 위함

## 2. 설정
- 기존 파일들은 묶어서 beforeProject로 backup
- next + react + typescript + tailwind를 기본
- shadcn 기반
- 시멘틱 마크업을 유지한 반응형 웹
- 웹 접근성 고려
- 다크모드 지원
- 기본적인 seo 지원
- 데이터는 server 에서 1회 호출 props로 전달. 5분 캐싱
- 메인 페이지
    - Frontend Developer, Publisher 직군을 기반으로 모든 것을 고려
    - 메인 페이지 설정
        - 프로필, 핵심 역량, 기술, 경력, 프로젝트
        - 스티키 메뉴 적용, 모바일 시 햄버거 메뉴
            - 왼쪽 직군
            - 오른쪽 기술, 경력, 프로젝트
    - 프로필
        - 직업에 기반한 간단한 자기소개 (이미지 포함)
    - 핵심 역량
        - 해당 직군에서 가져야할 작은 아이콘, 타이틀, 간단한 설명이 포함된 박스형 3가지 정도 나열
    - 기술
        - 해당 직군에 사용하는 기술들을 타입분류해서 나열
    - 경력
        - 박스형태 리스트 나열
            - 대제목 : 직군
            - 중제목 : 회사명
            - 소제목 : 기간
        - 박스 내 주요 업무내용 버튼 추가 해서 열고 닫기
        - 주요 업무 내용 영역은 제목, 내용 리스트 형태를 가지는 리스트
    - 프로젝트
        - 2가지 탭으로 구성 (FE, publish)
        - 시트 기준 앞 데이터 4개까지 우선 노출
        - 관련 내역 더보기 버튼 필요, 클릭 시 관련 리스트 페이지 이동
        - FE (현재 데이터 미구현 - 테스트 데이터로 페이지 구현(구글 시트 고려))
            - 회사, 타이틀, 간단 설명은 3줄 최대 3줄 정도 말줄임
            - 자세히 보기 버튼, 클릭 시 상세는 페이지 처리
            - 상세 페이지
                - [회사] 타이틀
                - 간단 설명 
                - 개발 기간, 투입 인원 표기
                - 본문은 위 내용과 구분자 처리
                - 트러블 슈팅 리스트 
                    - 문제, 해결 방안 한 묶음으로 구분되게
                - 성과
        - publish (현재 구글 시트 데이터)
            - marsony 스타일 페이지 리스트
            - lazy loading 적용
            - 현재 html 카드 마크업 형태 
            - 이미지는 최적화 하되 선명도 있게, 상단 정렬
